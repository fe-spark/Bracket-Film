# ❓ Bracket-Film 核心原理与排障手册 (FAQ)

本文档旨在深入阐述 **Bracket-Film** 的底层机制，帮助开发者与管理员理解系统如何处理海量影视数据并保持高度一致性。

---

## 1. 核心机制：主从架构 (Master-Slave)

### Q: 为什么必须设置“主站”才能正常工作？
**A**: 在多源聚合流程中，主站（Master）承担着**元数据骨架**的角色。
- **主数据（Master Data）**：主站负责写入核心影片元数据（即 `search_infos` 与 `movie_detail_infos`）。这些数据是前台搜索、分类导航及详情页展示的唯一基准。
- **采集逻辑隔离**：附属站（Slave）仅采集播放列表（`movie_playlists`）。如果没有主站提供骨架，附属站采集的数据将无法通过 `MovieKey` 挂载到具体影片上，导致“有源无内容”。

### Q: 切换主站会触发哪些原子操作？
**A**: 系统建立了严密的切换保护机制：
1. **单主站强制互斥**：设置新主站时，系统自动降级旧主站，确保全局唯一 Master。
2. **元数据重置 (`MasterFilmZero`)**：提升为 Master 会触发全局索引清空。
3. **播放源热挂载**：通过 `MovieSourceMapping` 全局 ID 映射技术，只要新主站的基础数据重建完成，原有附属站的播放源会基于指纹自动重新挂载，无需重新采集附属站。

## 2. 深度检索：联动筛选 (Faceted Search)

### Q: 联动筛选的底层逻辑是什么？
**A**: 系统弃用了低效的递归嵌套子查询，改用 **JOIN 扁平化查询逻辑**。
- **关系感应**：当您在“年份”点击 `2025` 时，系统会生成一个基于 `search_info` 与 `movie_tag_rel` 的 `INNER JOIN` 查询。
- **实时收缩**：其他维度（如“地区”）的选项会通过该交叉组合实时计算 `DISTINCT` 值。
- **性能支撑**：通过为 `MovieTagRel` 建立多维复合索引，即使在 50w+ 级关系数据下，也能实现毫秒级响应。

### Q: 为什么有时候筛选列表中会出现大类名称？
**A**: 这是由于部分采集源分类不规范（`Cid == Pid`）导致的。
- **逻辑修复**：我们已在 `BatchHandleSearchTag` 与 `GetTagsByTitle` 中加入了逻辑拦截。系统会自动识别并屏蔽掉那些与父类 ID 重合的冗余分类项，确保筛选列表只展示纯粹的二级分类。

## 3. 数据治理：孤儿清理与去重

### Q: 什么是“孤儿数据”，为什么要清理？
**A**: 孤儿数据指在 `movie_playlists` 中有播放源记录，但在当前主站检索表（`search_infos`）中找不到对应关联的“无主”记录。
- **产生原因**：主站切换、影片更名或采集源删除。
- **清理逻辑**：定时任务会扫描所有 `MovieKey`，若其无法映射到任何现有的主站内容指纹，则会被物理删除以释放空间并提高查询效率。

### Q: 系统如何实现内容级去重？
**A**: 依赖 **`ContentKey` (内容指纹)** 技术。
- **核心标识**：优先使用豆瓣 ID（`DbId`），若缺失则使用标准化的影片名哈希。
- **指纹冲突检测**：入库时如果发现指纹冲突，系统会自动执行 `Upsert` 逻辑，合并播放源并对齐元数据，而非生成重复记录。

## 4. 常见排障 (Troubleshooting)

### Q: 修改了配置或新入库数据后，为什么没有立即生效？
- **现状**：系统已实现 **全自动缓存同步**。
- **机制**：在影片采集入库（`BatchHandleSearchTag`）、分类状态调整或手动同步完成后，后端会自动清空 Redis 缓存（`ClearAllSearchTagsCache`）。
- **注意**：如果发现没有立即生效，可能是浏览器端的 HTTP 缓存或 CDN 缓存，请尝试强制刷新（Ctrl+F5）。系统层面的 Redis 缓存目前是实时更新的。

### Q: 视频详情页显示“暂无播放源”？
1. 确认该影片是否有关联的 `movie_playlist` 记录。
2. 确认主站的影片名与附属站的影片名是否差异过大（哈希匹配失效）。
3. 检查是否开启了主站采集但尚未完成。

### Q: Docker 环境下无法连接宿主机数据库？
- **解决**：在配置文件中使用 `host.docker.internal` 代替 `127.0.0.1`，并确保数据库用户的 Host 权限允许从容器网段连接。
