# 采集分类完整流程逻辑文档（重写版）

## 1. 当前版本结论

本项目分类链路采用“结构化映射优先 + 语义兜底”的双轨策略，核心目标是：

- 采集分类可稳定落到本地 `Pid/Cid`
- Category 筛选只展示真实子类，不展示大类本身
- Web 与 TVBox 使用同一套筛选数据源，行为一致
- 根分类顺序固定，不依赖数据库 `sort`

---

## 2. 全链路概览

分类数据从采集到展示共 5 段：

1. 采集站分类树抓取  
2. 分类映射落库（`category_mappings`）  
3. 影片详情归类（写入 `search_info.pid/cid`）  
4. 标签统计（写入 `search_tag_item`）  
5. Web / TVBox 按标签查询

---

## 3. 分类树采集与映射

### 3.1 入口

- 主采集流程先检查分类树，不存在则先同步分类树。
- 入口：`HandleCollect -> CollectCategory`

### 3.2 采集树抓取

- `spiderCore.GetCategoryTree` 从采集站 `ac=list` 解析 `class` 树。

### 3.3 映射落库（版本化 Upsert）

`SaveCategoryTree(sourceId, tree)` 关键逻辑：

- 每次同步生成一个 `mapping_version`
- 对每个来源 `type_id` 使用 `(source_id, source_type_id)` 做 Upsert
- 同步完成后删除该 source 下旧版本映射

这样避免“先删后建”窗口期，采集中读取映射更稳定。

### 3.4 顶级分类与子类映射规则

- 先把来源大类名推断为标准大类（电影/电视剧/动漫/综艺/纪录片/短剧/其他）
- 若来源大类名不等于标准大类名（如“日本动漫”），会在标准大类下建子类并映射到该子类
- 来源子类也会映射到对应本地子类

---

## 4. 影片入库时的分类决策

`ConvertSearchInfo(sourceId, detail)` 决策顺序：

1. 用来源 `detail.Cid` 查结构化映射，得到本地 `resolvedCid`
2. 若拿到 `resolvedCid`，通过 `GetRootId` 得到 `correctPid`
3. 若 `correctPid` 为空，再用来源 `detail.Pid` 查映射
4. 仍失败则按 `detail.CName` 做语义归桶推断标准大类
5. 若 `resolvedCid` 为空，执行 `resolveFallbackCid(pid, cName)`（仅查已有子类，不创建）
6. 最终兜底：`resolvedCid = correctPid`

说明：当前版本已经移除采集热路径自动建类副作用，保证采集过程不修改分类结构。

---

## 5. 标签生成与筛选来源

### 5.1 触发点

- `BatchSaveOrUpdate` / `SaveSearchInfo` 后触发 `BatchHandleSearchTag`

### 5.2 Category 标签策略（结构驱动）

- 优先使用 `Cid -> Category.Name` 反查名称生成 Category 标签
- 仅在反查失败时回退到 `info.CName`
- 过滤规则：`Category.value == pid` 视为大类本身，不入库

### 5.3 Upsert 规则

- 唯一键：`(pid, tag_type, value)`
- 冲突时：`score = score + 1`，并同步更新 `name`

---

## 6. 查询层逻辑（Web）

`GetSearchInfosByTags` 核心行为：

- 分类过滤严格模式：
  - `Cid` 是根类 -> 查 `pid = Cid`
  - `Cid` 是子类 -> 查 `cid = Cid`
  - 未传 `Cid` -> 查 `pid = Pid`
- Plot 使用 `LIKE`
- Area/Language/Year 使用等值查询
- “其他”(`__others__`) 通过排除热门 Top 值实现聚合

---

## 7. 查询层逻辑（TVBox）

### 7.1 配置与分类来源

- `/api/provide/vod` 返回 `class + filters + list`
- `class` 来自 `GetActiveCategoryTree`
- `filters` 来自 `GetSearchTag(Pid)`

### 7.2 一致性规则

TVBox 已与站内对齐：

- 支持 `year=__others__`
- 支持 `area/language/plot` 的 `__others__`
- `language` 与 `lang` 双参数兼容
- Plot 的“其他”排除数量限制与站内一致（最多 5 条）

---

## 8. 根分类顺序策略（重要更新）

当前根分类顺序不再依赖数据库 `sort`，由代码固定规则决定：

1. 电影  
2. 电视剧  
3. 动漫  
4. 综艺  
5. 纪录片  
6. 短剧  
7. 其他

该规则已同时应用到：

- 完整分类树 `GetCategoryTree`
- 活跃分类树 `GetActiveCategoryTree`

---

## 9. 常见故障排查

### 9.1 Category 为空

优先检查：

1. `category_mappings` 是否存在该 source 的映射  
2. `search_info` 中是否大量 `cid = pid`  
3. `search_tag_item` 中 `tag_type='Category'` 是否有多条不同 `value`

### 9.2 TVBox 与站内筛选结果不一致

优先检查：

1. 是否使用了旧进程（未重启）  
2. TVBox 请求参数是否为 `language` 或 `lang`  
3. `year/area/language/plot` 是否传入 `__others__` 并命中对应分支

---

## 10. 设计原则

- 结构化映射优先，语义兜底次之
- 分类结构修改只在“分类同步”阶段发生
- 采集热路径尽量无副作用
- 查询层严格 `Pid/Cid`，避免模糊包含
- Web 与 TVBox 共用一套标签数据，行为统一
